# Story 15-5: Lint Rule for Bare Exception Swallowing

## Status: verifying

## Story

As a developer,
I want code review to flag `except Exception: pass` (Python) and empty catch blocks,
So that error swallowing doesn't cause silent production incidents.

## Acceptance Criteria

- [ ] AC1: Given code-review subagent reviews Python code, when it finds `except Exception: pass`, then it flags it as HIGH severity <!-- verification: cli-verifiable -->
- [ ] AC2: Given `post-write-check.sh` hook runs after writing a Python file, when the file contains `except Exception: pass`, then it prints a warning <!-- verification: cli-verifiable -->

## Technical Notes

**Decision 7 (Error Handling).** This story extends error-handling enforcement beyond TypeScript to Python code reviewed or generated by codeharness.

### Semgrep Rule for Python (AC1)

Create a Semgrep rule in `patches/observability/` (or a new `patches/error-handling/` directory) that catches bare exception swallowing in Python:

```yaml
# patches/error-handling/no-bare-except.yaml
rules:
  - id: no-bare-except-pass
    pattern: |
      try:
        ...
      except Exception:
        pass
    message: "Bare `except Exception: pass` swallows errors silently. Handle the error, log it, or add a # IGNORE: comment explaining why."
    severity: ERROR
    languages: [python]
    metadata:
      category: error-handling
      confidence: HIGH

  - id: no-bare-except-ellipsis
    pattern: |
      try:
        ...
      except Exception:
        ...
    message: "Bare `except Exception: ...` swallows errors silently."
    severity: ERROR
    languages: [python]
```

The code-review subagent (in `src/modules/review/orchestrator.ts`) should include this Semgrep rule when reviewing Python files. The review module already runs Semgrep as part of its analysis -- add this rule to the rule set.

### Post-write Hook (AC2)

Create or update `hooks/post-write-check.sh` (Claude Code plugin hook) to check Python files after they're written:

```bash
# Check for bare except swallowing in Python files
if [[ "$FILE" == *.py ]]; then
  if grep -n "except Exception.*pass" "$FILE"; then
    echo "[WARN] Bare 'except Exception: pass' detected in $FILE"
    echo "       Handle the error or add a # IGNORE: comment"
  fi
fi
```

This hook fires in Claude Code after a file write, giving immediate feedback during development.

### Integration with Code Review

In `src/modules/review/orchestrator.ts`, ensure the Semgrep rule scan includes:
1. The new `patches/error-handling/` rules
2. Python files in the review scope
3. Severity mapping: Semgrep `ERROR` -> review `HIGH` severity finding
