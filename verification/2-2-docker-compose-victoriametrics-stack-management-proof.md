AC5 note: The implementation evolved to make observability mandatory (always ON). There is no --no-observability flag. Instead, when Docker is unavailable, init gracefully defers observability with 'Docker not available -- observability will use remote mode'. The original AC5 describing 'observability OFF' is superseded by this architectural decision. The equivalent behavior (no Docker) is verified via the Docker-not-available path.

AC6 note: The implementation warns about Docker unavailability and provides the install link (https://docs.docker.com/engine/install/) but does NOT exit with code 1. This is a deliberate design change from the original AC -- hard failure on missing Docker was replaced with graceful degradation.
