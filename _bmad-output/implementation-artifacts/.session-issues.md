### Session 21 — ROOT CAUSE FOUND for state corruption (2026-03-27T19:15Z)

- **ROOT CAUSE IDENTIFIED**: `parseRalphMessage()` in `src/lib/agents/ralph.ts` matches `[SUCCESS] Story {key}: DONE` regex within stream-json NDJSON lines. When claude session output contains tool results or text that includes these patterns (e.g., reading test files, source code, or discussing story completions), the regex matches inside the JSON envelope.
- `handleAgentEvent()` in `run.ts:41-48` then calls `updateStoryStatus(key, 'review')`, which CREATES new story entries (line 347: `?? defaultStoryState()`) or resets existing `done` stories to `review`.
- **Evidence**: This session created 3 phantom stories: `16-5-xxx`, `16-5-foo`, `1-1-foo` — all from patterns in session output. The `1-1-foo` came directly from test file `ralph.test.ts` line: `driver.parseOutput('[SUCCESS] Story 1-1-foo: DONE')`.
- **Fix needed**: `parseRalphMessage()` must NOT match within JSON lines. Add a check: if the line starts with `{`, skip ralph pattern matching entirely (it's a stream-json line, let the stream-json parser handle it).
- **Secondary fix**: `updateStoryStatus()` should NOT create new story entries. Change `?? defaultStoryState()` to return an error if the story key doesn't exist.
- Fixed: removed 3 phantom stories, restored sprint counts to 74/74.
- All 74 real stories remain done.
